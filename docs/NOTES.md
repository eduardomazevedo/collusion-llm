# Project Notes

## Sample definitions (current pipeline)
- **Main analysis dataset**: `data/datasets/main_analysis_dataset.feather` built by `src/post_query/analysis/main_dataset.py` using queried transcripts (`data/datasets/queries.sqlite`) merged with transcript details, Compustat, human ratings, LLM flags, validation flags, and human audit flags.
- **Benchmark sample**: transcripts present in `data/datasets/human_ratings.csv`; `benchmark_human_flag` is true when `joe_score >= 50` or `acl_manual_flag == 1` (thresholds in `config.py`).
- **LLM-flagged**: earliest `SimpleCapacityV8.1.1` query per transcript with score `>= 75` (gpt-4o-mini); IDs in `data/intermediaries/top_transcripts.csv`; details in `data/datasets/top_transcripts_data.csv`.
- **LLM-validated**: transcripts with 11 queries (original + 10 repeats) whose `mean_score_ten_repeats >= 75`; flag stored as `llm_validation_flag` in `main_analysis_dataset`.
- **Human-audit flags**: `human_audit_sample`/`human_audit_flag` merged from `assets/human_audit_v3.xlsx` (see below).

## Human audit sample asset (`assets/human_audit_v3.xlsx`)
- Static Excel file; not generated by any repo script (read by `main_dataset.py`, `benchmarking_analysis.py`, and audit-related analysis scripts).
- Expected columns: `transcript_id` and `T/F/N` (N treated as non-collusive, same as prior `U`).
- Other columns in the file are currently ignored by the pipeline.

## Joe/U-Haul threshold comparison
- Source: `data/datasets/human_ratings.csv` (rows with non-missing `joe_score`, n = 91).
- Threshold 50 (used in code): 5 flagged as collusive, 86 non-collusive.
- Threshold 75 (hypothetical): 1 flagged as collusive, 90 non-collusive.
